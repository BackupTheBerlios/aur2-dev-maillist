I haven't looked at how Django permissions work exactly, but looking  
at the _user_can_delete_package filter, it seems it's dependent on  
package.__unicode__ returning the package name. I think it should use  
package.name instead.

The last conditional is confusing. It makes me ask the question what  
happens if the user isn't a moderator. It seems it won't return  
anything if the package exists, so the user gets a blank page? The  
conditional should be removed imo. Also the try/except is redundant  
since the get_object_or_404 function already redirects to a 404 error  
page. It is thus guaranteed that the except block will never execute.
